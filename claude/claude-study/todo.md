## 권한 모드 (Permission Modes)

`Shift+Tab`으로 순환하거나 `/config`에서 변경

| 모드                  | 자동 실행 범위                     | 특징                      |
|---------------------|------------------------------|-------------------------|
| `default`           | 읽기만                          | 기본값. 쓰기/실행은 모두 물어봄      |
| `acceptEdits`       | 읽기 + 파일 편집 + 기본 파일 명령어       | 코드 수정은 자동, 큰 명령은 물어봄    |
| `plan`              | 읽기만 (default와 동일)            | 실행 없이 계획만 제안함           |
| `auto`              | 거의 모든 것 (내부 안전 분류기 사용)       | 긴 작업에 적합, 프롬프트 피로 감소    |
| `dontAsk`           | `permissions.allow`에 등록된 도구만 | CI/스크립트용. 비허가 도구는 자동 거부 |
| `bypassPermissions` | 모든 것 (보호 경로 제외)              | 격리된 컨테이너 전용. 위험!        |

## 노력 수준 (Effort Levels)

`/effort` 명령어로 변경

| 레벨       | 속도/비용       | 적합한 상황               |
|----------|-------------|----------------------|
| `low`    | 가장 빠름/저렴    | 단순하고 짧은 작업           |
| `medium` | 빠름/보통       | 비용 절감이 우선일 때         |
| `high`   | 보통          | 일반적인 균형점             |
| `xhigh`  | 느림/비쌈       | 대부분의 코딩/에이전트 작업 최적   |
| `max`    | 가장 느림/가장 비쌈 | 깊은 추론 필요 시 (세션 내 한정) |

> 모델별로 동일 레벨이라도 실제 동작이 다름 (Opus의 `high` ≠ Sonnet의 `high`)

## 빠른 모드 (Fast Mode)

`/fast` 토글

| 항목         | 내용                                           |
|------------|----------------------------------------------|
| 대상 모델      | Opus 4.6 전용 (다른 모델로 전환 아님)                   |
| 속도         | 약 2.5배 빠름                                    |
| 비용         | 더 비쌈                                         |
| 적합한 상황     | 빠른 반복 작업, 실시간 디버깅                            |
| Effort와 관계 | 독립적으로 조합 가능 (`/fast` + `low` effort = 최대 속도) |

## 출력 스타일 (Output Styles)

`/config` → Output style

| 스타일           | 동작                                 |
|---------------|------------------------------------|
| `default`     | 일반 소프트웨어 엔지니어링 모드                  |
| `explanatory` | 구현 중간중간 "인사이트" 제공                  |
| `learning`    | `TODO(human)` 마커로 사용자가 직접 코딩하도록 유도 |
| custom        | 직접 정의한 마크다운 시스템 프롬프트               |

---

**핵심 구분:**

- **Permission mode** → Claude가 *무엇을 실행할 수 있는지*
- **Effort level** → Claude가 *얼마나 깊이 생각하는지*
- **Fast mode** → *속도/비용 트레이드오프*
- **Output style** → Claude가 *어떻게 응답하는지*

카테고리별로 정리하고 실무 활용 사례까지 포함해서 정리해드립니다.

## 세션 관리

| 명령어        | 설명                    | 실무 활용 사례                      |
|------------|-----------------------|-------------------------------|
| `/clear`   | 대화 초기화 (컨텍스트 비움)      | 작업이 끝나고 새 기능 개발 시작 전          |
| `/compact` | 대화 압축 (선택적 포커스 지정 가능) | 긴 세션 후 토큰 절약하며 계속 작업          |
| `/rename`  | 세션 이름 변경              | 여러 프로젝트를 오가며 작업할 때 구분         |
| `/resume`  | 이전 세션 재개              | 어제 하던 PR 리뷰 작업 이어하기           |
| `/branch`  | 현재 시점에서 대화 분기         | "이 방향으로 구현하면 어떨까?" 실험해보고 싶을 때 |
| `/rewind`  | 이전 시점으로 되돌리기          | 잘못된 방향으로 코드 수정됐을 때 롤백         |
| `/exit`    | 세션 종료                 | -                             |

## 모델 & 성능 설정

| 명령어       | 설명                                   | 실무 활용 사례                          |
|-----------|--------------------------------------|-----------------------------------|
| `/model`  | 모델 변경                                | 복잡한 아키텍처 설계엔 Opus, 빠른 수정엔 Sonnet  |
| `/effort` | 추론 깊이 설정 (low/medium/high/xhigh/max) | 단순 린팅은 `low`, 복잡한 버그 디버깅은 `xhigh` |
| `/fast`   | 빠른 모드 토글 (Opus 4.6 전용, 2.5배 빠름)      | 라이브 페어 프로그래밍 세션, 빠른 Q&A           |
| `/config` | 설정 UI (테마, 모델, 출력 스타일 등)             | 프로젝트마다 다른 출력 스타일 설정               |

## 실무 활용 시나리오 예시

**신규 프로젝트 온보딩:**

```
/init → /memory → /permissions
```

CLAUDE.md 생성 → 팀 규칙 저장 → 자주 쓰는 명령어 자동 허용

**PR 머지 전 체크리스트:**

```
/diff → /security-review → /ultrareview
```

변경사항 확인 → 보안 검토 → 심층 코드 리뷰

**긴 세션 관리:**

```
/context → /compact "현재 버그 수정에 집중" → /cost
```

토큰 상태 확인 → 압축 → 비용 체크

**CI 자동화:**

```
/autofix-pr
```

백그라운드에서 CI 실패 감시 → 자동 수정 → 푸시
